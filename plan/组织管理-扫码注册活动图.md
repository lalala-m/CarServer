### 5.1 扫码注册完整时序

```mermaid
sequenceDiagram
    participant Admin as 管理员

        Admin->>Web: 3. 点击「生成二维码」
        Web->>API: 4. POST /api/qr_code/generate
        API->>API: 5. 生成 qr_token (UUID)
        API->>API: 6. 计算过期时间
        API->>DB: 7. INSERT qr_code_register
        DB-->>API: 8. 返回 qr_id
        API->>API: 9. 使用 ZXing 生成二维码图片
        API->>API: 10. 保存图片到服务器
        API-->>Web: 11. 返回 qr_url, qr_token
```

## 六、批量导入时序图

### 6.1 Excel 批量导入时序

```mermaid
sequenceDiagram
    participant Admin as 管理员
    participant Web as Web管理端
    participant API as 后端API
    participant DB as 数据库
    participant Excel as Excel文件

    rect rgb(200, 230, 200)
        Note over Admin,Excel: 阶段一：准备导入数据
        Admin->>Web: 1. 选择角色类型
        Admin->>Web: 2. 点击「下载导入模板」
        Web->>API: 3. GET /api/import/download_template?role_type=xxx

        Web-->>Admin: 5. 下载模板
        Admin->>Admin: 6. 填写 Excel 数据
        Admin->>Web: 7. 上传填写好的 Excel
    end

    rect rgb(200, 200, 240)
        Note over Web,DB: 阶段二：数据解析与校验
        Web->>API: 8. POST /api/import/excel<br/>(file, role_type)
        API->>API: 9. 使用 POI 解析 Excel
        API->>API: 10. 遍历每一行数据
        loop 每行数据校验
            API->>API: 11. 检查必填字段
            API->>API: 12. 校验手机号格式
            API->>API: 13. 校验工号唯一性
            API->>API: 14. 检测重复行
        end
        API->>DB: 15. 预查询需要的数据
        DB-->>API: 16. 返回查询结果
    end

    rect rgb(240, 200, 200)
        Note over API,DB: 阶段三：批量插入（事务）
        API->>API: 17. 开启事务
        loop 逐行插入
            API->>DB: 18. INSERT 角色数据
            alt 插入成功
                DB-->>API: 19. 返回成功
                Note over API: 计数成功数
            else 插入失败
                DB-->>API: 20. 返回错误
                Note over API: 记录失败行及原因
            end
        end
        API->>API: 21. 提交事务
        API-->>Web: 22. 返回导入结果<br/>(total, success, failed, failed_list)
        Web-->>Admin: 23. 展示导入结果统计
        alt 有失败行
            Admin->>Admin: 24. 查看失败详情
            Admin->>Admin: 25. 修正数据后重新导入
        else 全部成功
            Admin->>Admin: 26. 流程结束
        end
    end
```
