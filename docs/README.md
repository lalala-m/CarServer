## 🎯 功能概述

### 实现目标

为车友服务系统新增**个人二维码与扫码注册**功能，支持：

| 角色 | 数据库表 | 扫码注册 | 状态 |
|------|----------|----------|------|
| 会员 | `owner_user` | ✅ 支持 | 复用已有表 |
| 财务审核 | `financial_user` | ✅ 支持 | 复用已有表 |
| 信息员 | `info_user` | ✅ 支持 | ❌ 新建 |
| 业务顾问 | `business_consultant` | ✅ 支持 | ❌ 新建 |

### 新建数据库表（共 3 张）

| 表名 | 用途 |
|------|------|
| `qr_code_register` | 二维码追踪表 |
| `info_user` | 信息员表 |
| `business_consultant` | 业务顾问表 |

## ❓ 常见问题

**Q: 业务顾问和商家用户有什么区别？**

A: `business_user`（商家用户）是系统中已有的表，用于管理商家/门店信息。`business_consultant`（业务顾问）是新建的表，用于管理 4S 店的业务接待人员。这是两个不同的角色。

**Q: 为什么需要 `qr_code_register` 表？**

A: 为了实现扫码注册流程，需要追踪每个二维码的 Token、状态、有效期等信息。现有表中的 `two_dimensional_code` 字段只存储图片路径，不支持这些功能。

**Q: 二维码有效期是多久？**

A: 默认 30 天，可在生成时自定义。
